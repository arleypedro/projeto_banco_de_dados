Introdução aos conceitos de bancos de dados

  1. O que é um Banco de Dados?

    2. Um banco de dados é uma coleção organizada de informações, ou dados, que se relacionam entre si.
    Imagine uma biblioteca com várias estantes, cada uma contendo livros de um determinado assunto.
    Essa organização facilita a busca e recuperação de informações específicas.
    Da mesma forma, um banco de dados armazena dados de forma estruturada, permitindo consultas e análises eficientes.

  1. Por que usar um Sistema de Gerenciamento de Banco de Dados (SGBD)?

    2. Um SGBD é um software que permite criar, acessar, gerenciar e controlar um banco de dados.
    Utilizar um SGBD traz diversas vantagens, como:

    Compartilhamento de Dados:

    • Permite que múltiplos usuários acessem e utilizem os mesmos dados de forma simultânea.

    Segurança:

    • Controla o acesso aos dados, garantindo que apenas usuários autorizados possam visualizá-los ou modificá-los.

    Consistência de Dados:

    • Impõe regras para garantir que os dados armazenados sejam válidos e confiáveis.

    Backup e Recuperação:

    • Facilita a criação de cópias de segurança e a recuperação dos dados em caso de falhas.

  1. Modelos de Dados:

    Existem diferentes formas de organizar os dados em um banco.
    Veremos os principais modelos:

      2. Relacional:

    • O modelo mais utilizado atualmente. Os dados são organizados em tabelas com linhas e colunas, como em uma planilha.
    • Hierárquico: Os dados são organizados em uma estrutura similar a uma árvore genealógica, com um único nó raiz e ramificações.
    • Em Rede: Similar ao modelo hierárquico, mas permite relacionamentos mais complexos entre os dados.
    • Orientado a Objetos: Combina dados e operações em entidades chamadas objetos.

  1. Usuários e Administradores de Banco de Dados

    2. Um banco de dados possui diferentes tipos de usuários:

    Usuários Finais:

    • Aqueles que utilizam o banco de dados para realizar tarefas específicas, como consultar informações ou inserir novos registros.

    Administradores de Dados:

    • Responsáveis por projetar, implementar e manter o banco de dados, garantindo sua segurança e performance.

    Definições, e responsabilidades dos diferentes tipos de usuários em um banco de dados.

  1. Arquitetura de um SGBD

    2. Explorando a Arquitetura e Abstração de Dados

    Introdução:

    Vamos explorar a arquitetura de um Sistema de Gerenciamento de Banco de Dados (SGBD) e entender como os diferentes níveis de abstração de dados contribuem para a organização e manipulação eficiente das informações.

    Níveis de Abstração de Dados:

    Os níveis de abstração de dados são um conceito fundamental em SGBDs, permitindo aos usuários interagir com o banco de dados de diferentes maneiras, ocultando a complexidade da implementação física dos dados.

    • Nível Físico: O nível mais baixo de abstração descreve como os dados são fisicamente armazenados em disco. Detalhes como estrutura de arquivos, organização de índices e métodos de acesso são definidos neste nível.
    • Nível Lógico (ou Conceitual): Define a estrutura lógica do banco de dados, incluindo tabelas, relacionamentos entre elas, restrições de integridade e outras informações relevantes.
    • Nível de Visão (ou Externo): O nível mais alto de abstração, que oferece diferentes visões personalizadas do banco de dados para os usuários. Cada visão apresenta apenas os dados relevantes para um grupo específico de usuários, simplificando a interação e protegendo informações confidenciais.

    Independência de Dados

    A independência de dados é um dos principais benefícios da utilização de níveis de abstração
    A independência de dados permite modificar o esquema de um nível sem afetar os níveis superiores.

    • Independência de Dados Lógica: A capacidade de modificar o esquema conceitual sem afetar os programas de aplicação ou as visões externas.
    • Independência de Dados Física: A capacidade de modificar o esquema físico (armazenamento) sem afetar o esquema conceitual.

    Linguagens de Banco de Dados Para interagir com o SGBD, utilizamos linguagens de banco de dados. As principais categorias são:

    • DDL (Linguagem de Definição de Dados): Utilizada para definir o esquema do banco de dados. Comandos como CREATE, ALTER e DROP pertencem a esta categoria.
    • DML (Linguagem de Manipulação de Dados): Usada para manipular os dados armazenados no banco. Comandos como SELECT, INSERT, UPDATE e DELETE fazem parte desta categoria.
    • DCL (Linguagem de Controle de Dados): Define os privilégios de acesso e segurança do banco de dados. Comandos como GRANT e REVOKE são exemplos.

    Conclusão

    Compreender a arquitetura de um SGBD e os diferentes níveis de abstração de dados é essencial para o desenvolvimento e gerenciamento eficiente de banco de dados. A independência de dados e as linguagens de banco de dados são ferramentas que facilitam a administração, a segurança e a flexibilidade do sistema.

  1. Modelagem de Dados Entidade-Relacionamento (ER) é uma ferramenta essencial para o projeto de bancos de dados.

      2. Conceitos Básicos:

    • Entidades: Uma entidade representa um objeto do mundo real com características em comum.
      Por exemplo, em um sistema de biblioteca, "Livro" e "Usuário" seriam entidades.
    • Atributos: Características que descrevem uma entidade.
      "Título", "Autor" e "ISBN" seriam atributos da entidade "Livro".
    • Relacionamentos: Associações entre entidades.
      O relacionamento "Empréstimo" poderia conectar as entidades "Livro" e "Usuário".

      • Diagramas ER e suas Notações

      Diagramas ER: São representações gráficas do modelo de dados.
      Utilizam símbolos para representar entidades, atributos e relacionamentos, facilitando a visualização e a comunicação do projeto do banco de dados.

      Notação: Existem diferentes notações para diagramas ER.

      • "Pé de Galinha" utiliza-se para representar relacionamentos uma seta com tres linhas .
        Indica a cardinalidade do relacionamento (1:1, 1:N, N:M)
      • Retângulos: Representam entidades.
      ◦ Elipses: Representam atributos. ◦ Losangos: Representam relacionamentos. ◦ Linhas: Conectam entidades a seus atributos e relacionamentos. Mapeamento do Modelo ER para o Modelo Relacional O modelo ER é convertido para o modelo relacional para ser implementado em um SGBD relacional. • Entidades: Cada entidade se torna uma tabela no banco de dados. • Atributos: Os atributos da entidade se tornam colunas da tabela. • Relacionamentos: Dependendo da cardinalidade (um-para-um, um-para-muitos, muitos-para-muitos), o relacionamento pode gerar: ◦ Uma nova tabela com chaves estrangeiras. ◦ Chaves estrangeiras em tabelas existentes.
    Exemplo: • Entidade "Livro": Tabela "Livros" com colunas "Título", "Autor", "ISBN". • Entidade "Usuário": Tabela "Usuários" com colunas "Nome", "Endereço", "CPF". • Relacionamento "Empréstimo": Tabela "Empréstimos" com chaves estrangeiras para "Livros" e "Usuários", além de atributos como "Data do Empréstimo" e "Data da Devolução". Conclusão A Modelagem de Dados ER é fundamental para o projeto de bancos de dados eficientes e bem estruturados. Compreender seus conceitos e o processo de mapeamento para o modelo relacional é crucial para a criação de sistemas de informação robustos.
